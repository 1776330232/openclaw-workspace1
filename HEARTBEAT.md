# 心跳检查清单

每次心跳按顺序检查，有事就报，没事回 HEARTBEAT_OK。

## 检查项

1. **网关健康**
   - 命令：`curl.exe -s -o NUL -w "%{http_code}" --noproxy 127.0.0.1 http://127.0.0.1:18789/`
   - 正常：返回 200
   - 异常：非 200 → 报警，建议用户重启网关

2. **记忆落盘**
   - 检查：今天的 `memory/YYYY-MM-DD.md` 是否存在且有内容
   - 正常：文件存在，今天有至少1条记录
   - 异常：文件不存在或为空 → 提醒落盘

3. **待办检查**
   - 检查：读 `tasks.json`，有没有 status="待做" 且创建超过3天还没动的任务
   - 正常：无此类任务，或任务正在进行
   - 异常：有积压任务 → 报给 Hang

4. **僵尸任务检查**
   - 检查：读 `tasks.json`，有没有 `updatedAt` 超过 **3 天**，且 status 仍处于进行中 / 待处理的任务
   - 正常：无此类任务，或任务近期有推进
   - 异常：有僵尸任务 → 报给 Hang，并标记需要 orchestrator 决定是继续、blocked 还是关闭

5. **孤儿 SESSION-STATE 检查**
   - 检查：`SESSION-STATE*.md` 有没有超过 **1 天** 未更新，且没有对应 active task（进行中 / 待处理）
   - 正常：无孤儿文件，或文件仍对应活跃任务
   - 异常：有孤儿文件 → 报给 Hang，提示清理或收口

6. **记忆提炼**
   - 检查：读今天的每日日志，有没有标记了"跨天有效"但还没进 MEMORY.md 的内容
   - 正常：已落盘，或没有需要落盘的内容
   - 异常：有遗漏 → 立刻写入

7. **学习手册**
   - 检查：读今天的每日日志，有没有 `[新能力]` 标注但还没更新到 `docs/LEARNING_HANDBOOK.md`
   - 正常：已更新，或没有新能力
   - 异常：有遗漏 → 立刻更新

8. **错误日志回顾**（每周一次，不是每次心跳）
   - 检查：`.learnings/ERRORS.md` 最近7天有没有新增错误
   - 如果有：提炼出通用规律，看有没有需要更新到 AGENTS.md 或 MEMORY.md 的教训

9. **日志归档**（心跳独占，cron 不负责归档）
   - 检查：`memory/` 下有没有超过 7 天的日志文件
   - 有的话：蒸馏内容进 MEMORY.md（替换/新增，不追加重复），然后移入 `memory/archive/`
   - 归档前先检查 `memory/archive/` 是否存在，不存在则创建

## 用户离线模式（动态调频）

- 检查 `memory/YYYY-MM-DD.md` 中最后一条记录的时间
- 如果 Hang 超过 **1小时** 没有新消息：
  - **只做整理，不推任务**：
    - 补记今天遗漏的日志
    - 提炼跨天有效内容进 MEMORY.md
    - 回顾 `.learnings/ERRORS.md`
  - **禁止**：主动推进 tasks.json 中的任务（除非任务明确标记 `heartbeat_ok: true`）
  - 然后回 HEARTBEAT_OK
- 如果 Hang 刚刚活跃（< 1小时）：跳过额外工作，正常检查即可

## 安静规则

- 深夜（23:00-08:00）只检查网关（第1条），其他全部跳过（含归档、错误回顾、离线分支）
- 上次检查 < 30 分钟，回 HEARTBEAT_OK
- 没有任何异常，回 HEARTBEAT_OK
- 有多项异常，合并成一条消息报告，不要逐条发
