## [ERR-20260310-001] 给 cron agent turn 设短超时导致假死循环

**时间**：2026-03-09 ~ 2026-03-10 01:05
**行为**：给 bookmark-watcher cron 手动设了 `timeoutSeconds: 120`（后改 300），导致 cron runs 持续记录 timeout error，但底层 session 实际跑完并产出结果。表现为"明明超时了但又出结果了"的矛盾现象。
**根因**：OpenClaw cron 用 `Promise.race` 实现超时。计时器先到 → 调度器记 error → 但 abort signal 是协作式的（只在 turn 间检查），不能中断 mid-turn 的 LLM/browser 调用 → session 继续跑完。而 OpenClaw 对 agent turn 类任务的默认安全超时是 **60 分钟**（`AGENT_TURN_SAFETY_TIMEOUT_MS`），手动设短值覆盖了这个合理默认。
**正确做法**：agent turn 类 cron 任务不要设显式 `timeoutSeconds`，让它用默认的 60 分钟安全值。除非你有明确理由且清楚 Promise.race + cooperative abort 的行为。
**修法**：从 `jobs.json` 删除 `timeoutSeconds` 字段 + 清零 `consecutiveErrors` + 重启网关。
**额外教训**：排障时先查源码确认机制，不要拍脑袋调参数。从 120→300 的盲改浪费了时间。
