### 01:05：bookmark-watcher cron 根因定位 + 修复完成
- **根因**：`executeJobCoreWithTimeout` 用 `Promise.race` 实现超时。超时计时器先到 → 调度器记 error → 但底层 session 不会被真正杀死（abort signal 是协作式，只在 turn 间检查，不能中断 mid-turn 的 LLM/browser 调用）→ session 继续跑完 → announce 结果
- **加剧因素**：`timeoutSeconds` 被手动设为 120（后改 300），覆盖了 OpenClaw 默认的 agent turn 安全超时（60 分钟 = `AGENT_TURN_SAFETY_TIMEOUT_MS`）
- **修法**：从 `jobs.json` 删除 `timeoutSeconds` 字段，回退到默认 60 分钟；同时清零 `consecutiveErrors` 避免退避延迟
- **验证**：`openclaw cron list` 显示 status=ok，enabled=true，下一轮 13 分钟后自动触发
- **经验**：OpenClaw agent turn 类 cron 任务不应设短超时；默认 60 分钟安全值是设计好的，除非有明确理由不要覆盖
